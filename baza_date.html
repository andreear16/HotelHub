Model conceptual al bazei de date

Entitate Descriere Atribute principale
User Reprezintă toți utilizatorii aplicației (clienți, angajați, administratori) id_user, nume, email, telefon, parola, rol, data_inregistrare
Camera Cameră disponibilă în hotel id_camera, numar_camera, tip_camera, pret_noapte, disponibilitate
Rezervare Rezervare efectuată de un client și procesată de un angajat id_rezervare, FK id_user_client, FK id_user_angajat, FK id_camera, data_checkin, data_checkout, status_rezervare
Factura Document financiar emis pentru o rezervare id_factura, FK id_rezervare, data_emitere, total, metoda_plata
Serviciu Servicii suplimentare oferite clienților id_serviciu, denumire, pret
Rezervare_Serviciu Legătură M:N între rezervare și servicii FK id_rezervare, FK id_serviciu

Relațiile dintre entități

  • User – Rezervare (1 la N)

    Un utilizator cu rol de client poate efectua mai multe rezervări. Un utilizator cu rol de angajat (recepționer) poate gestiona mai multe rezervări. Fiecare rezervare este asociată unui singur client și unui singur angajat. Relația este implementată prin două chei externe (id_user_client și id_user_angajat) în tabelul Rezervare.

  • Camera – Rezervare (1 la N)

    O cameră poate fi rezervată de mai multe ori în timp, dar o rezervare se referă la o singură cameră. Relația permite gestionarea disponibilității camerelor și prevenirea rezervărilor simultane.

  • Rezervare – Factură (1 la 1)

    Fiecare rezervare confirmată generează o singură factură. Factura conține informațiile financiare corespunzătoare rezervării respective. Relația este implementată prin cheia externă id_rezervare din tabelul Factura.

  • Rezervare – Serviciu (M la N)

    O rezervare poate include mai multe servicii suplimentare (ex: mic dejun, spa, room service). Un serviciu poate fi asociat mai multor rezervări. Relația este implementată prin tabelul intermediar Rezervare_Serviciu, care conține cheile externe id_rezervare și id_serviciu.

  • User – Factură (1 la N)

    Un utilizator cu rol de angajat (recepționer) poate emite mai multe facturi. Fiecare factură este emisă de un singur angajat. Relația este implementată prin cheia externă id_user_angajat din tabelul Factura.
